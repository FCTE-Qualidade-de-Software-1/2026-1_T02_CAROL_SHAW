# Evidência da M5

Esta página registra a análise estática utilizada no cálculo da **M5 - Condensabilidade**.

## 1. Escopo

A unidade de análise foi cada arquivo de código de produção capaz de declarar dependências internas.

| Subsistema | Caminhos incluídos | Extensões |
| :--------- | :----------------- | :-------- |
| Frontend | `no_fluxo_frontend_svelte/src/` | `.ts`, `.svelte` |
| Backend | `no_fluxo_backend/src/` | `.ts` |
| Serviços Python | `DBA/`, `mcp_agent/`, `no_fluxo_backend/parse-pdf/` e `no_fluxo_backend/start_and_monitor.py` | `.py` |

<p align="center">Tabela 1 - Escopo da análise da M5. Fonte: Equipe Carol Shaw, 2026.</p>

Foram excluídos:

- testes e arquivos de declaração `.d.ts`;
- dependências de bibliotecas e frameworks;
- código gerado e diretórios de dependências;
- documentação, SQL e configurações sem imports de módulos;
- scripts de implantação e exemplos presentes na documentação de Kubernetes.

## 2. Ferramentas

| Ferramenta | Versão | Aplicação |
| :--------- | :----- | :-------- |
| Madge | 8.0.0 | Grafo e verificação do backend TypeScript |
| svelte2tsx | 0.7.56 | Conversão estática dos scripts Svelte para representação TypeScript |
| TypeScript Compiler API | 5.9.3 | Resolução dos imports, reexports e imports dinâmicos do frontend |
| Pydeps | 3.0.6 | Grafo de imports dos módulos Python |

<p align="center">Tabela 2 - Ferramentas da análise estática. Fonte: Equipe Carol Shaw, 2026.</p>

O Madge não identificou os imports declarados dentro dos arquivos `.svelte`. Para evitar que esses componentes fossem classificados incorretamente como independentes, seus scripts foram convertidos com `svelte2tsx` e analisados pela API do compilador TypeScript.

Alguns arquivos Python possuem nomes incompatíveis com a resolução de módulos do Pydeps, como `01_insert_...`, e o diretório `parse-pdf` contém hífen. A ferramenta foi executada sobre uma cópia temporária com esses nomes normalizados. O conteúdo dos módulos e as declarações de importação não foram alterados.

## 3. Regra de Classificação

Para cada componente \(C\), foi construído o conjunto \(D(C)\) de módulos internos importados diretamente.

- **Não dependente:** \(D(C) = \varnothing\);
- **Dependente:** \(D(C) \neq \varnothing\).

Imports externos foram desconsiderados porque não representam dependência entre componentes do produto. Imports de tipo, reexports e imports dinâmicos internos foram contabilizados.

## 5. Ciclos

Foram identificados três grupos cíclicos no frontend:

1. `lib/types/curso.ts`, `lib/types/equivalencia.ts` e `lib/types/materia.ts`;
2. `lib/components/ui/dialog/dialog-content.svelte` e `lib/components/ui/dialog/index.ts`;
3. `lib/components/ui/scroll-area/scroll-area.svelte` e `lib/components/ui/scroll-area/index.ts`.

O Madge não encontrou ciclos nos 20 módulos do backend TypeScript. Os ciclos não alteram diretamente o cálculo da M5, mas evidenciam acoplamento bidirecional.

## 7. Cálculo

$$
M5 = \frac{\text{componentes não dependentes}}{\text{total de componentes}} \times 100
$$

$$
M5 = \frac{107}{287} \times 100 = 37{,}28\%
$$

Conforme os critérios da Fase 2, o resultado é **inaceitável**, pois está abaixo de 50%.
