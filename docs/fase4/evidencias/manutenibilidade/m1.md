# Evidência da M1

## 1. Procedimento

O levantamento foi elaborado exclusivamente a partir do diretório [`no_fluxo_frontend_svelte/src/lib/components`](https://github.com/unb-mds/2025-1-NoFluxoUNB/tree/ba6db878b9dfa36fb034916612c4cf58ddf43475/no_fluxo_frontend_svelte/src/lib/components), no commit [`ba6db878b9dfa36fb034916612c4cf58ddf43475`](https://github.com/unb-mds/2025-1-NoFluxoUNB/commit/ba6db878b9dfa36fb034916612c4cf58ddf43475).

Foram aplicadas as seguintes regras de contagem:

1. componentes Svelte independentes foram contados por arquivo;
2. componentes internos de uma família em `components/ui/<familia>/` foram agrupados pelo módulo público `index.ts`;
3. arquivos fora da pasta `components` não foram incluídos no levantamento;
4. um componente foi considerado reutilizado quando importado por dois ou mais arquivos de contexto distintos;
5. imports internos de uma família de UI não foram considerados contexto externo.

### 1.1 Evidência em vídeo

O vídeo [`svelte_uso.mp4`](../../../assets/videos/svelte_uso.mp4) demonstra o uso da extensão Svelte no ambiente de desenvolvimento para navegar entre os arquivos de componentes e visualizar, arquivo por arquivo, quais trechos do código os referenciam. Esse recurso auxiliou a identificação dos contextos de importação considerados no levantamento.
