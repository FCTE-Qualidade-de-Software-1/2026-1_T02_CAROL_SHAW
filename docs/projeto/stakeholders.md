# Requisitante e Partes Interessadas

Esta seção identifica os atores que influenciam e são impactados pela avaliação de qualidade do NoFluxoUNB.

| Stakeholder | Papel | Necessidades e Expectativas | Influência na Avaliação |
| :--- | :--- | :--- | :--- |
| **Estudantes da UnB** | Usuário Final | Precisão nas informações de pré-requisitos e fluxograma sempre disponível. | Define a prioridade da característica de **Confiabilidade**. |
| **Equipe de Desenvolvimento** | Fornecedor | Facilidade em adicionar novos currículos de cursos e corrigir bugs rapidamente. | Define a necessidade de **Manutenibilidade**. |
| **Coordenadores de Curso** | Requisitante / Stakeholder | Garantir que o software reflita fielmente os projetos pedagógicos (PPC). | Exige rigor na integridade dos dados (Adequação Funcional). |
| **Professora da Disciplina** | Avaliador Externo | Validar se os processos de medição seguem as normas de qualidade. | Define os critérios de sucesso do relatório final. |
